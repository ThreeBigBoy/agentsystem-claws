# 联动更新检查清单

## 一句话定义

执行全局检查时的逐项检查清单，确保变更后6类文档全部同步更新，防止遗漏任何关联组件。

## 使用场景

### 必须使用

| 场景 | 说明 |
|-----|------|
| **Step 10 全局检查** | 复盘完成后自动触发 |
| **用户追问遗漏** | 用户问「是否遗漏」「是否完成」 |
| **大规模变更后** | 涉及规则层、多宿主、多技能升级 |

### 禁止使用（不要过度检查）

| 场景 | 说明 |
|-----|------|
| **仅修改单个脚本** | 影响范围明确且有限 |
| **仅修改文档格式** | 无其他组件依赖 |

## 检查清单结构

```
联动更新检查清单（6类文档）
├── 第1类：根级文档（2项）
├── 第2类：宿主SOP（4项）
├── 第3类：Agent文档（2项）
├── 第4类：全局规则（2项）
├── 第5类：Skill文档（2项）
└── 第6类：Memory文档（3项）
    总计：15项检查点
```

## 逐项检查清单

### 第1类：根级文档检查（影响新用户第一印象）

| # | 检查项 | 检查内容 | 通过标准 | 常见遗漏 |
|---|--------|---------|---------|---------|
| 1.1 | README.md | 更新日志是否记录本次变更 | ✅ 最新变更已记录 | 只改代码不改README |
| 1.2 | 新用户快速开始.md | 质量保障机制章节是否与最新规范一致 | ✅ 8+1/10步闭环描述一致 | 升级闭环后未同步 |

### 第2类：宿主SOP检查（影响新用户初始化体验）

| # | 检查项 | 检查内容 | 通过标准 | 常见遗漏 |
|---|--------|---------|---------|---------|
| 2.1 | Cursor SOP | `platform-adapters/cursor/从0开始*.md` 是否同步 | ✅ 质量保障机制章节一致 | 只更新Cursor，遗漏其他宿主 |
| 2.2 | VS Code SOP | `platform-adapters/vscode/从0开始*.md` 是否同步 | ✅ 质量保障机制章节一致 | 只更新Cursor，遗漏VS Code |
| 2.3 | Continue SOP | `platform-adapters/generic/continue-从0开始*.md` 是否同步 | ✅ 质量保障机制章节一致 | 遗漏Continue |
| 2.4 | OpenAI-Codex SOP | `platform-adapters/generic/openai-codex-从0开始*.md` 是否同步 | ✅ 质量保障机制章节一致 | 遗漏OpenAI-Codex |

### 第3类：Agent文档检查（影响角色定义一致性）

| # | 检查项 | 检查内容 | 通过标准 | 常见遗漏 |

### 第4类：全局规则检查（影响治理层一致性）

| # | 检查项 | 检查内容 | 通过标准 | 常见遗漏 |

| 4.2 | projects-rules-for-agent.md | 是否需要更新 | ✅ 相关规则已同步 | 更新skills-rules后未检查projects-rules |

### 第5类：Skill文档检查（影响技能执行标准）

| # | 检查项 | 检查内容 | 通过标准 | 常见遗漏 |

## 关键规则

### 规则1：不得声称「全局检查完成」直到全部通过

**错误示例**:
```
用户：全局检查完成了吗？
Agent：✅ 全局检查完成（实际上只检查了2类文档）
```

**正确做法**:
```
用户：全局检查完成了吗？
Agent：我执行联动更新检查清单，逐项检查15项...
[逐项检查]
结果：发现4项遗漏，正在修复...
[修复完成]
重新验证...
✅ 15项全部通过，现在可以声称「全局检查完成」
```

### 规则2：用户追问后立即执行全面检查

**触发条件**:
- 用户问「是否遗漏」「是否完成」「是否同步」
- 用户追问「那XXX呢」「YYY是否更新」

**必须执行**: 无论之前是否声称完成，立即执行全面检查

### 规则3：发现遗漏立即修复后再声称

**错误示例**:
```
Agent：发现4个宿主SOP未更新
用户：那怎么办？
Agent：下次记得更新（未立即修复）
```

**正确做法**:
```
Agent：发现4个宿主SOP未更新，立即执行更新...
[更新完成]
Agent：已修复，重新验证中...
[验证完成]
✅ 全部通过
```

## 检查执行流程

```
开始全局检查
    │
    ├──→ 执行变更影响分析
    │       └──→ 识别影响范围（6类文档中哪些受影响）
    │
    ├──→ 逐项执行15项检查
    │       ├──→ 已更新？→ 标记通过
    │       └──→ 未更新/需确认？→ 标记待修复
    │
    ├──→ 汇总检查结果
    │       ├──→ 全部通过？→ 声称「全局检查完成」
    │       └──→ 存在遗漏？→ 进入修复循环
    │
    ├──→ 修复循环（如需要）
    │       ├──→ 执行遗漏项更新
    │       ├──→ 重新验证
    │       └──→ 通过？→ 声称完成 / 不通过？→ 继续修复
    │
    └──→ 记录到迭代日志
```

## 常见遗漏模式

| 遗漏模式 | 示例 | 预防措施 |
|---------|------|---------|
| **宿主遗漏** | 只更新Cursor，遗漏VS Code | 检查清单强制检查4个宿主 |
| **版本号格式** | skills-rules用v2026-03-17 | 检查清单强制检查vX.Y格式 |
| **术语表遗漏** | 新增概念未补充 | 检查清单强制检查术语表 |
| **Agent角色遗漏** | 更新主Agent未更新子Agent | 检查清单强制检查子Agent |
