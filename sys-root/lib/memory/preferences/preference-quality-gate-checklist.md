# 8+1 质量闭环统一质量门禁检查清单

## 使用说明

**强制要求**：
- 每个阶段出口必须逐项检查，不可跳过
- 只有当前阶段所有检查项通过，才能进入下一阶段
- 检查过程须记录到迭代日志

**检查流程**：
```
阶段执行完成
    ↓
执行本阶段质量门禁检查清单
    ↓
全部通过？──→ 是 → 进入下一阶段
    ↓ 否
修复问题
    ↓
重新执行该阶段（含重新检查）
    ↓
通过 → 进入下一阶段
```

### 关键判定规则（重要）

| 全局检查结论 | 是否可以视为「完整闭环」| 后续动作 |
|-------------|----------------------|---------|
| **10.1-10.20 全部通过** | ✅ **可以** | 本次 change-id 完整闭环完成 |
| **发现遗漏但未修复** | ❌ **不可以** | 立即修复遗漏项 → 重新验证 → 全部通过 |
| **用户追问后发现遗漏** | ❌ **不可以** | 执行全面检查 → 修复 → 重新验证 |

### 准出判定
- **✅ 通过**：10.1-10.20 全部通过 → 本次 change-id **真正完整闭环完成**
- **❌ 不通过**：任一项不通过 → 完成遗漏步骤 → 重新验证

**重要声明**: 
- 不得声称「全局检查完成」直到 10.1-10.20 **全部**通过
- 用户追问「是否遗漏」「是否完成」时，必须执行全面检查
- 发现遗漏立即修复后再声称完成
