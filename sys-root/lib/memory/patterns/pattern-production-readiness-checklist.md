# 生产就绪检查清单（安全性、稳定性、可观测性）

## 一句话定义

系统上线前的生产就绪检查清单，通过安全性、稳定性、可观测性三大模块的检查，确保系统能够安全、稳定、可观测地运行。

## 问题背景

### 常见的生产就绪遗漏

```
┌─────────────────────────────────────────────────────────────┐
│                    常见的生产就绪遗漏                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  遗漏 1: 安全性模块                                            │
│  • 缺少输入验证 → XSS/SQL注入风险                               │
│  • 缺少权限控制 → 未授权访问                                     │
│  • 缺少审计日志 → 无法追踪操作                                  │
│                                                              │
│  遗漏 2: 稳定性模块                                            │
│  • 缺少限流 → 流量冲击导致系统崩溃                               │
│  • 缺少熔断 → 级联故障                                          │
│  • 缺少降级 → 失败时无优雅处理                                   │
│                                                              │
│  遗漏 3: 可观测性模块                                          │
│  • 缺少日志 → 问题排查困难                                       │
│  • 缺少指标 → 无法监控系统状态                                   │
│  • 缺少追踪 → 无法定位问题                                       │
│                                                              │
│  遗漏 4: 边界条件处理                                          │
│  • 超大输入处理                                                  │
│  • 资源限制（磁盘、内存）                                        │
│  • 超时处理                                                      │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 模式解决方案

### 生产就绪三大模块

### 生产就绪检查清单

```markdown
## 生产就绪检查清单

### 一、安全性检查

#### 1.1 输入验证
- [ ] 长度限制（最大输入长度10KB）
- [ ] 格式验证（正则表达式）
- [ ] 危险模式过滤
  - [ ] XSS脚本过滤 `<script>...</script>`
  - [ ] SQL注入过滤 `DROP TABLE`, `DELETE FROM`
  - [ ] 危险命令过滤 `rm -rf /`
  - [ ] 代码注入过滤 `__import__`, `eval`, `exec`
- [ ] 敏感字符转义（HTML标签转义）

#### 1.2 权限控制
- [ ] 角色定义（owner/maintainer/developer/viewer）
- [ ] 权限矩阵
  - [ ] owner: read/write/confirm/approve/delete
  - [ ] maintainer: read/write/confirm/approve
  - [ ] developer: read/write/confirm
  - [ ] viewer: read
- [ ] 变更级权限检查
- [ ] 操作前权限验证

#### 1.3 审计日志
- [ ] 关键操作记录（确认、审批、降级）
- [ ] 日志格式（JSONL）
- [ ] 日志内容
  - [ ] 时间戳
  - [ ] 操作类型
  - [ ] 用户身份
  - [ ] 输入哈希（隐私保护）
  - [ ] 操作结果
- [ ] 日志存储（.audit目录）

### 二、稳定性检查

#### 2.1 限流机制
- [ ] 全局限流（100 req/s，burst 200）
- [ ] 变更级限流（20 req/s，burst 50）
- [ ] 用户级限流（10 req/s，burst 30）
- [ ] 令牌桶算法实现
- [ ] 限流白名单（/health, /docs等）

#### 2.2 熔断机制
- [ ] 熔断器实现（CircuitBreaker）
- [ ] 三状态机（CLOSED/OPEN/HALF_OPEN）
- [ ] 失败阈值（默认5次）
- [ ] 成功阈值（默认3次）
- [ ] 熔断超时（默认60秒）
- [ ] 自动恢复能力
- [ ] 熔断状态监控端点

#### 2.3 重试策略
- [ ] 指数退避（base_delay * 2^attempt）
- [ ] 最大重试次数（默认3次）
- [ ] 最大延迟限制（默认30秒）
- [ ] 重试间隔可配置

#### 2.4 降级方案
- [ ] 降级触发条件
- [ ] 最小化产出物生成
- [ ] 降级标记（⚠️ 降级生成）
- [ ] 降级告警通知

### 三、可观测性检查

#### 3.1 链路追踪
- [ ] TraceContext实现
- [ ] RequestID生成（UUID）
- [ ] RequestID透传（HTTP Header）
- [ ] SpanID层级追踪
- [ ] 性能计时（start_time, end_time）
- [ ] 标签系统（tags）

#### 3.2 指标采集
- [ ] QPS指标
- [ ] 延迟指标（P50/P95/P99）
- [ ] 错误率指标
- [ ] 资源使用率（CPU/内存/磁盘）
- [ ] 指标采集中间件

#### 3.3 日志记录
- [ ] 结构化日志（JSON）
- [ ] 日志级别（DEBUG/INFO/WARNING/ERROR）
- [ ] 日志字段
  - [ ] timestamp
  - [ ] level
  - [ ] message
  - [ ] request_id
  - [ ] trace_id
  - [ ] user_id
  - [ ] duration_ms
- [ ] 日志采样配置
- [ ] 日志聚合方案

#### 3.4 健康检查
- [ ] /health端点
- [ ] 存活探针（Liveness）
- [ ] 就绪探针（Readiness）
- [ ] 依赖检查
  - [ ] 数据库连接
  - [ ] 缓存连接
  - [ ] 外部服务
- [ ] 健康状态返回

### 四、边界条件检查

#### 4.1 资源限制
- [ ] 磁盘空间检查（最低1GB）
- [ ] 内存检查（最低500MB）
- [ ] CPU使用率监控
- [ ] 文件句柄限制

#### 4.2 超大输入处理
- [ ] 文件大小检查（默认50MB）
- [ ] Git diff截断处理（默认10MB）
- [ ] 大文件分片处理
- [ ] 流式处理支持

#### 4.3 超时处理
- [ ] 请求超时设置
- [ ] 连接超时设置
- [ ] 操作超时设置
- [ ] 超时优雅降级
```

## 实施建议

### 实施步骤

1. **阶段一: 安全性**（优先级P0）
   - 实现SecurityValidator
   - 实现AuthChecker
   - 实现AuditLogger

2. **阶段二: 稳定性**（优先级P0）
   - 实现RateLimiter
   - 实现CircuitBreaker
   - 实现RetryPolicy
   - 实现FallbackManager

3. **阶段三: 可观测性**（优先级P0）
   - 实现TraceContext
   - 实现指标采集
   - 实现结构化日志
   - 实现健康检查

4. **阶段四: 集成测试**（优先级P1）
   - 集成到AgentInvoker
   - 集成到FastAPI中间件
   - 集成到Step节点

### 验收标准

| 模块 | 验收标准 | 测试方法 |
|-----|---------|---------|
| 安全性 | 通过渗透测试 | 模拟攻击测试 |
| 稳定性 | 通过压力测试 | 1000并发，99.9%成功率 |
| 可观测性 | 通过监控验证 | 查看日志、指标、追踪 |
