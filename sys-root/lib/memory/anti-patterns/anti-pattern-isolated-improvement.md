# 孤立改进反模式 (Anti-Pattern: Isolated Improvement)

## 一句话定义

只更新直接相关文档，忽略关联文档，导致系统各组件不一致的反模式。

## 典型症状

| 症状 | 示例 |
|-----|------|
| **用户困惑** | "为什么README写的是v1.3，SOP里还是v1.2？" |
| **版本混乱** | 同一概念在不同文档中版本号不一致 |
| **执行偏差** | 按SOP执行，结果与最新规范不符 |
| **反复追问** | 用户不断问「那XXX呢」「YYY是否更新」 |

## 危害

| 危害类型 | 具体表现 |
|---------|---------|
| **用户体验差** | 新用户按SOP初始化，发现与README不一致 |
| **执行标准不一** | 不同Agent看到不同版本，执行结果不一致 |
| **信任损失** | 用户发现遗漏后会质疑「是否还有其他遗漏」 |
| **返工成本** | 发现问题后需要补更新，甚至重新验证 |

## 根因分析

### 为什么会产生孤立改进？

| 根因 | 说明 |
|-----|------|
| **局部视角** | 「我更新了核心文档，入口文档应该自动同步」 |
| **惯性思维** | 「以前都是只改一处，从没出过问题」 |
| **检查缺失** | 没有「联动更新检查清单」逐项验证 |
| **急于求成** | 「先声称完成，遗漏的下次再补」 |

### 5 个 Why 分析

```
问题：为什么全局更新后仍有遗漏？

Why 1: 为什么只更新了核心文档，没更新宿主SOP？
答案：我以为「新用户快速开始.md」更新了，宿主SOP就会自动同步

Why 2: 为什么会有这种错误假设？
答案：我没有意识到宿主SOP是独立的初始化入口文档

Why 3: 为什么没有识别出这种依赖关系？
答案：缺乏「变更影响分析」步骤，没有系统性地识别所有受影响文档

Why 4: 为什么没有执行变更影响分析？
答案：没有这个步骤的显性要求，也没有检查清单强制执行

Why 5: 为什么没有建立这个机制？
答案：之前没有认识到「全局检查与联动更新」是完整闭环的必需环节

根因：缺乏「变更影响分析」机制和「联动更新检查清单」
```

## 如何避免

### 立即执行（本次避免）

| # | 行动 | 执行标准 |
|---|------|---------|
| 1 | 执行变更影响分析 | 列出所有可能受影响的文档类型 |
| 2 | 使用联动更新检查清单 | 逐项检查15项，全部通过才能声称完成 |
| 3 | 修复发现的遗漏 | 立即修复，不得拖延到「下次」 |
| 4 | 重新验证 | 修复后必须重新验证全部通过 |

### 长期预防（机制建设）

| # | 机制 | 说明 |
|---|------|------|
| 1 | **Step 10 全局检查** | 将「全局检查与联动更新」加入完整闭环 |
| 2 | **变更影响分析 Pattern** | 沉淀 pattern-change-impact-analysis |
| 3 | **联动更新检查清单** | 使用 preference-coordinated-update-checklist |
| 4 | **术语表自动链接** | 确保新增概念同步到术语表 |

### 检查清单（执行前自查）

```markdown
## 孤立改进反模式检查

在声称「全局检查完成」前，确认：

- [ ] 已执行变更影响分析
- [ ] 已使用联动更新检查清单逐项验证
- [ ] 15项检查全部通过
- [ ] 没有「下次再补」的遗留项
- [ ] 已记录到迭代日志

如果以上任一项未勾选，**不得声称**「全局检查完成」
```

## 修复方法

### 如果发现自己是孤立改进

```
Step 1: 立即停止声称「全局检查完成」
Step 2: 执行联动更新检查清单，全面扫描遗漏
Step 3: 列出所有遗漏项
Step 4: 立即执行同步更新
Step 5: 重新验证全部通过
Step 6: 记录到迭代日志
Step 7: 反思：为什么会遗漏？
Step 8: 沉淀 anti-pattern-isolated-improvement 到 Memory
```

### 如果发现他人是孤立改进

```
Step 1: 明确指出「这是孤立改进反模式」
Step 2: 引用本 anti-pattern 文档
Step 3: 建议执行联动更新检查清单
Step 4: 协助识别遗漏项
Step 5: 验证修复结果
```

## 真实案例

### 案例：8+1 升级 v1.3 遗漏 4 个宿主 SOP

**时间**: 2026-03-17  
**事件**: 将 8+1 质量闭环升级为 10 步闭环  
**孤立改进**: 只更新了 pattern-complete-quality-closed-loop.md 和新用户快速开始.md  
**遗漏**: 未更新 4 个宿主 SOP（Cursor/VS Code/Continue/OpenAI-Codex）  
**发现**: 用户追问「既然认同，那是否文档都更新完了，再反思下！」  
**修复**: 立即执行全面检查，发现遗漏并修复  
**沉淀**: 创建本 anti-pattern 和联动更新检查清单
