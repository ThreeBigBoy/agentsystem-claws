# 反模式：术语定义漂移（Terminology Drift）

## 一句话定义

**将规范术语按日常理解或历史惯性执行，而非查阅规范中的明确定义，导致执行偏差和关键步骤遗漏。**

---

## 问题表现

### 典型症状

| 症状 | 说明 | 案例 |
|-----|------|------|
| **执行不完整** | 遗漏规范定义的关键步骤 | "归档"只标记完成，未合并 specs/ |
| **与规范不符** | 执行结果符合直觉但违反规范 | 设计文档更新但未移动 changes/ |
| **用户追问** | 用户发现执行偏差并追问 | 用户两次追问"是否真正归档" |
| **重复犯错** | 同一类术语反复出现理解偏差 | 多个规范术语存在类似问题 |

### 诊断信号

- 执行后用户追问"是否按规范执行？"
- 执行结果表面完成但缺少关键产出物
- 依赖"之前这么做"而非"规范这么说"
- 无法准确引用规范原文说明执行步骤

---

## 根本原因

### 1. 日常语言 vs 规范术语

| 术语 | 日常理解 | OpenSpec 定义 | 偏差后果 |
|-----|---------|--------------|---------|
| **归档** | 标记完成 | 合并 specs/ + 移动 changes/ | 规范未更新，目录未清理 |
| **评审** | 看看对不对 | 9项检查+结论+纪要+修复循环 | 评审不完整，无留痕 |
| **验收** | 测试通过 | 对照 PRD 逐项验证 | 验收不全面，遗漏验收项 |

### 2. 惯性思维依赖

```
错误推理链：
"上次归档是这么做的" → "这次也这么做" → "标记完成=归档"
         ↑                                    ↓
         └────────── 忽略规范升级 ───────────┘
```

### 3. 规范查阅缺失

- 执行前未查阅规范文档
- 未识别操作中的规范术语
- 未建立术语-规范链接

---

## 危害

| 危害类型 | 具体表现 | 严重程度 |
|---------|---------|---------|
| **执行偏差** | 遗漏关键步骤，操作不完整 | 高 |
| **质量不达标** | 表面完成但不符合规范 | 高 |
| **规范失效** | 规范定义与执行脱节 | 高 |
| **用户信任损失** | 需多次追问才能正确执行 | 中 |
| **知识无法沉淀** | 错误执行无法形成正确模式 | 中 |

---

## 避免方法

### 方法 1: 术语规范检查（Terminology Check）

**触发时机**: 任何涉及规范术语的操作

**执行步骤**:
1. **识别术语**: 识别操作中的规范术语（归档、评审、验收、提案等）
2. **查阅定义**: 查阅该术语在 OpenSpec 或相关规范中的明确定义
3. **对比理解**: 对比自己的理解与规范定义是否一致
4. **按规执行**: 严格按规范定义执行，而非日常理解

**检查清单**:
- [ ] 我是否识别了所有涉及的规范术语？
- [ ] 我是否查阅了这些术语的规范定义？
- [ ] 我的理解与规范定义是否一致？
- [ ] 我是否按规范定义执行了所有步骤？

### 方法 2: 建立术语-规范链接

**操作前强制查阅**:
```
执行"归档"前 → 查阅 OpenSpec.md 归档章节
执行"评审"前 → 查阅对应 skill 的评审定义
执行"验收"前 → 查阅 func-test skill 定义
```

### 方法 3: 惯性思维阻断

**质疑惯性**:
- 不问"之前怎么做"，问"规范怎么说"
- 每次执行前假设"规范可能已升级"
- 将"查阅规范"作为执行的第一步

## 案例库

### 案例 1: OpenSpec 归档执行缺陷（2026-03-16）

**场景**: check-langgraph-backend Step 8 归档

**错误表现**:
- 将"归档"理解为"标记 tasks.md 完成 + 更新 design.md"
- 遗漏真正的归档操作：合并 specs/ 规范 + 移动 changes/ 目录
- 用户两次追问后才发现问题

**根因**: 术语定义漂移 + 惯性思维

**避免**: 执行前查阅 OpenSpec.md 归档章节，明确归档操作步骤

### 案例 2: 评审结论处理偏差

**场景**: 代码评审结论为"有条件通过"

**错误表现**:
- 将"有条件通过"理解为"可以进入下一阶段"
- 未执行修复→重新评审→通过的循环

**根因**: 术语理解偏差

**避免**: 查阅 prd-review/architecture-review skill 中的结论处理定义

## 检查清单（防止术语定义漂移）

**执行任何规范术语操作前，必须完成**:

- [ ] 识别操作涉及的所有规范术语
- [ ] 查阅每个术语在规范中的明确定义
- [ ] 对比自己的理解与规范定义
- [ ] 按规范定义列出完整执行步骤
- [ ] 逐项执行并检查
- [ ] 完成后验证是否符合规范要求
