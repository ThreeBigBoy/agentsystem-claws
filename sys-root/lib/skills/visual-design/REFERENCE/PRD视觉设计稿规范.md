# PRD视觉设计稿规范

本规范定义在**Step 1 需求分析环节**，当项目涉及前端开发时，由 `visual-design` 技能产出的**交互视觉设计稿**的内容结构与质量标准。

---

## 一、定位

**交互视觉设计稿**是 PRD 产物的补充，用于：
1. **视觉化表达交互逻辑**：让开发/测试能直观理解用户操作流程
2. **定义视觉要素**：统一字体、颜色、布局尺寸、圆角等规范
3. **列出占位切图**：便于开发准备图片资源

**注意**：本规范产出的是"设计稿说明文档"而非"设计稿图片"，实际设计稿图片由设计工具产出，但本规范定义了需要涵盖的内容要素。

---

## 二、产出物结构

产出物存放于 `docs/project-prd-changes/[change-id]/visual-design/`，包含以下文件：

| 文件 | 必须/可选 | 说明 |
|------|---------|------|
| `交互逻辑.md` | 必须 | 页面流程、交互状态、用户操作路径 |
| `视觉要素规范.md` | 必须 | 字体、颜色、间距、圆角、阴影等 |
| `占位切图清单.md` | 可选（若涉及图片资源） | 图片资源清单、尺寸、格式 |

## 三、交互逻辑.md（必须）

### 3.1 页面结构与跳转

```markdown
## 页面结构

| 页面 | 路由/路径 | 说明 |
|------|---------|------|
| 首页 | / | 主入口 |
| 详情页 | /detail/:id | 查看详情 |

## 页面跳转流程

[A] --> [B] --> [C]
      ↘ [D]
```

### 3.2 交互状态定义

| 元素 | 状态 | 说明 |
|------|------|------|
| 按钮 | default / hover / active / disabled | 交互反馈 |
| 输入框 | default / focus / error / disabled | 输入反馈 |
| 列表项 | default / selected / loading | 选中状态 |
| 卡片 | default / hover | 悬停效果 |

### 3.3 关键用户流程

```markdown
## 关键流程：下单支付

1. 用户进入商品详情页
2. 点击"立即购买"按钮
3. 进入确认订单页（展示商品信息、地址、支付方式）
4. 点击"提交订单"
5. 进入支付页（展示支付二维码）
6. 支付成功后跳转订单完成页

## 异常流程

- 库存不足：提示"库存不足，请返回重选"
- 支付超时：展示倒计时，超时后提示"订单已取消"
```

## 四、视觉要素规范.md（必须）

### 4.1 字体系统

```markdown
## 字体层级

| 层级 | 字体 | 字号 | 字重 | 行高 | 使用场景 |
|------|------|------|------|------|---------|
| H1 | 系统默认 | 24px | 700 | 1.3 | 页面标题 |
| H2 | 系统默认 | 18px | 600 | 1.4 | 区块标题 |
| Body | 系统默认 | 14px | 400 | 1.6 | 正文内容 |
| Caption | 系统默认 | 12px | 400 | 1.5 | 辅助说明 |

## 字体规范

- 优先使用系统字体栈：-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial
- 中文字体：PingFang SC, Microsoft YaHei
- 英文/数字字体：SF Pro Text, Roboto
```

### 4.2 颜色系统

```markdown
## 主色板

| 色值 | 用途 | 使用场景 |
|------|------|---------|
| #007AFF | 主色 | 按钮、链接、图标 |
| #5856D6 | 次要色 | 辅助强调 |
| #34C759 | 成功色 | 成功提示、通过状态 |
| #FF3B30 | 错误色 | 错误提示、失败状态 |
| #FF9500 | 警告色 | 警告提示 |

## 中性色板

| 色值 | 用途 |
|------|------|
| #000000 | 主文字 |
| #333333 | 正文文字 |
| #666666 | 次要文字 |
| #999999 | 占位文字 |
| #E5E5EA | 分割线 |
| #F2F2F7 | 背景色 |
| #FFFFFF | 卡片背景 |
```

### 4.3 间距系统

```markdown
| 名称 | 数值 | 使用场景 |
|------|------|---------|
| xs | 4px | 紧凑元素间距 |
```

```markdown
## 圆角规范

| 元素 | 圆角值 | 说明 |
|------|-------|------|
```

### 4.5 阴影系统

```markdown
## 阴影规范

| 名称 | 值 | 使用场景 |
|------|---|---------|
| sm | 0 1px 2px rgba(0,0,0,0.1) | 轻微浮起 |

| lg | 0 8px 24px rgba(0,0,0,0.15) | 弹窗、浮层 |
```

### 4.6 动效规范

```markdown
## 动效参数

| 类型 | 时长 | 缓动函数 | 使用场景 |
|------|------|---------|---------|
| 微交互 | 200ms | ease-out | 按钮悬停、点击反馈 |
| 页面切换 | 300ms | ease-in-out | 页面转场 |
| 展开收起 | 250ms | ease-out | 折叠面板、展开详情 |
| 加载 | - | - |骨架屏/Loading |
```

## 六、质量标准

### 6.1 必须项

- [ ] 交互逻辑.md：包含页面跳转流程、关键用户流程
- [ ] 视觉要素规范.md：包含字体/颜色/间距/圆角定义
- [ ] 视觉要素与 PRD 功能需求对应，无遗漏

### 6.2 验收标准

- 开发可依据本文档还原80%以上的视觉设计
- 测试可依据本文档进行视觉回归测试
- 设计新增/修改时，只需更新本文档，无需重新口头描述
