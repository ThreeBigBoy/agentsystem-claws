# 数据分析技能（data-analysis）

## 触发场景

当用户请求以下类型时，自动激活本技能：

- 「分析这份数据」「帮我分析这个Excel文件」
- 「生成数据分析报告」「做个数据分析」
- 「创建可视化图表」「做个数据看板」
- 上传Excel/CSV文件并要求分析

## 核心能力

1. **数据读取**：支持Excel(.xlsx/.xls)、CSV文件
2. **数据清洗**：处理缺失值、异常值、数据类型转换
3. **多轮分析**：逐层深入分析，发现问题后下钻
4. **异常检测**：识别异常值并追溯原因
5. **分析沉淀**：每步分析输出Markdown文档
6. **可视化生成**：生成ECharts可视化网页

## 多轮分析流程

### 第一轮：数据概览分析

1. 读取数据，了解数据结构和字段类型
2. 输出数据概览（行数、列数、字段类型、缺失值情况）
3. **输出文字分析** → **沉淀到 `analysis_report.md`**

### 第二轮：核心指标分析

1. 计算核心KPI指标（总和、均值、最大值、最小值等）
2. 识别TOP N表现（最高/最低的类别）
3. **输出文字分析** → **追加到 `analysis_report.md`**

### 第三轮：分布与对比分析

1. 按分类维度进行分组分析
2. 计算各类别占比和排名
3. **输出文字分析** → **追加到 `analysis_report.md`**

### 第四轮：趋势分析（如有时间字段）

1. 按时间维度分析趋势变化
2. 计算环比/同比增长率
3. **输出文字分析** → **追加到 `analysis_report.md`**

### 第五轮：异常检测与下钻分析

1. 识别异常值（超出正常范围的数据）
2. **对异常值进行下钻分析**，追溯异常原因
3. 分析异常数据的特征（时间、类别、来源等）
4. **输出文字分析** → **追加到 `analysis_report.md`**

### 第六轮：生成可视化看板

1. 根据分析结果选择合适的图表类型
2. 生成ECharts可视化HTML文件
3. **输出最终分析总结** → **追加到 `analysis_report.md`**

## 分析输出格式

### 每轮分析输出要求

每完成一轮分析，必须：

1. **在对话中输出文字分析**：简洁说明发现了什么
2. **沉淀到Markdown文件**：将分析结果追加到 `analysis_report.md`

### Markdown文件格式

```markdown
# 数据分析报告

## 1. 数据概览
- 数据维度：[行数]行 × [列数]列
- 时间范围：[时间字段范围，如无则填"无时间字段"]
- 缺失值情况：[各字段缺失情况]

## 2. 核心指标分析
- 核心KPI1：[指标名]：[数值] [单位]
- 核心KPI2：[指标名]：[数值] [单位]
- 核心KPI3：[指标名]：[数值] [单位]
- TOP表现：[最高项]：[数值] / [最低项]：[数值]

## 3. 分布与对比分析
- [维度1]分布：[占比最高的类]贡献[X]%，排名第一
- [维度2]分布：[主要发现]
- 对比结论：[核心对比发现]

## 4. 趋势分析（如有时间字段）
- 整体趋势：[上升/下降/平稳]，最[快/慢]增速期：[时间]
- 周期性：[是否存在周期波动，如周末效应、季节性等]
- 异常波动：[如有，发生在哪个时间点]

## 5. 异常检测与下钻分析
### 发现的异常
- 异常1：[描述]，数值[偏离正常范围]

### 下钻分析
- 时间维度：[异常发生在哪个时间粒度]
- 类别维度：[异常集中在哪个类别]
- 特征维度：[异常数据有何共同特征]
- 可能原因：[分析推断]

## 6. 分析总结
1. [核心发现1]
2. [核心发现2]
3. [核心发现3]
```

## 异常值下钻分析规范

发现异常值时，必须进行下钻分析：

### 1. 识别异常

- **数值异常**：超出正常波动范围（如：超出均值±N倍标准差，N视业务而定）
- **趋势异常**：环比/同比变化超过业务容忍阈值（如：超过50%，阈值视指标特性而定）
- **分布异常**：某类别占比突然变化超过容忍范围（如：超过20%，阈值视业务而定）
- **空值/缺失**：关键字段出现缺失或null

### 2. 下钻维度

- **时间维度**：异常发生在哪个时间粒度（年/季度/月/周/日/小时）
- **分类维度**：异常集中在哪个分类变量（如：地区/产品线/用户类型）
- **数值特征**：异常的数值范围、集中区间、偏离程度
- **组合维度**：异常是否在特定组合条件下出现（如：某地区+某产品）

### 3. 原因追溯

- 分析异常数据的共同特征
- 对比正常数据与异常数据的差异
- 给出可能的原因假设

### 4. 输出结论

- 异常现象描述
- 影响范围评估
- 可能原因分析
- 建议措施

## 可视化看板生成

### CDN资源引用

生成HTML时必须引用以下CDN资源：

```html
<head>
    <meta charset="UTF-8">
    <title>数据分析看板</title>
    <!-- ECharts核心库（版本按需选择） -->
    <script src="https://cdn.jsdelivr.net/npm/echarts@5.4.3/dist/echarts.min.js"></script>
    <!-- ECharts词云扩展（如需词云图表） -->
    <script src="https://cdn.jsdelivr.net/npm/echarts-wordcloud@2.1.0/dist/echarts-wordcloud.min.js"></script>
</head>
```

> **注意**：如无法访问外部CDN，需改为本地引用或其他可用的图表库。

### 最终输出要求

1. **3个KPI指标卡**：展示最重要的3个数值指标
2. **3个核心结论**：简洁有力的数据发现
3. **多个可视化图表**：根据数据特征选择合适的图表类型

### HTML文件要求

- **必须引用ECharts CDN**
- **如使用词云，必须引用词云CDN**
- 响应式设计，适配不同屏幕
- 现代化UI风格
- 图表支持交互（tooltip、缩放等）

## 图表选择指南

### 对比分析类

| 图表类型 | ECharts类型 | 适用场景 | 示例 |
|----------|-------------|----------|------|

| 分组柱状图 | bar (多系列) | 多维度对比 | 两个以上维度的对比 |

## 核心原则

1. **多轮深入**：逐层深入分析，不浅尝辄止
2. **异常下钻**：发现异常必须追溯原因
3. **实时沉淀**：每轮分析都输出文字并沉淀到Markdown
4. **数据驱动**：所有结论必须有数据支撑
5. **可视化优先**：用图表直观展示数据
6. **CDN必须引用**：确保ECharts及扩展库正确加载
