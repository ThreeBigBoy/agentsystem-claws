# 实现完成自检

> 前端/后端 Agent 执行 coding-implement 完成本轮实现后，须按本清单自检，确保代码与 OpenSpec、specs、project 约定一致，任务与元信息文档已更新，可进入 code-review 与 func-test。  
> 本技能主要产出为**代码**及可选的 `info-database/`、`info-service-interface/` 与 tasks 更新；本节约定**实现完成后的自检清单**，不强制新增独立文档结构。

---

## 一、自检清单（实现完成后必过）

在完成当前 change-id 下的编码实现（及可选的信息文档更新）后，按下列项自检；任一项为否时，补全再视为实现完成。

| # | 自检项 | 说明 |
|---|--------|------|
| 1 | **change-id 锁定** | 是否始终围绕单一 change-id 实现？未混入其他变更范围？ |
| 2 | **与 specs 对应** | 每条已实现的 Requirement 是否有可识别的代码单元（函数/方法/组件）对应？注释或提交信息中是否可引用 Requirement/Scenario？ |
| 3 | **与 project/project-rules 一致** | 目录、文件命名、分层、依赖方向是否符合 `openspec/project.md` 与 `project-rules/`？是否未在错误层次实现逻辑（如业务逻辑落在表现层）？ |
| 4 | **tasks 已更新** | 是否已在 `openspec/changes/[change-id]/tasks.md` 中将本轮完成的实现任务勾选为 `[x]`？纯实现类任务可在实现或 code review 后勾选；验证类任务须由验收方勾选。 |
| 5 | **info-database / info-service-interface** | 若本次变更涉及新增或调整数据库表、对外服务接口，是否已在项目根目录 `info-database/`、`info-service-interface/` 下补充或更新说明与脚本？ |
| 6 | **无硬编码与违规** | 是否无硬编码密钥、敏感路径、违反 projects-rules 安全与命名规则？ |
| 7 | **可被 review 与测试** | 代码是否已保存、变更范围是否明确，可供 code-review 与 func-test 使用？ |

---

## 二、与技能流程的对应

- **coding-implement 步骤 4～5 之后**：完成编码与文件组织、可选的信息文档更新后，执行本节「一、自检清单」；通过后再进入 code-review 或直接交付测试。
- **与 code-review、func-test**：自检通过后的实现，作为 code-review 与 func-test 的输入；若 review 或测试发现问题，按 tasks 与对应技能流程闭环。
