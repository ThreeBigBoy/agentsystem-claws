# 后端表现层编码规范（API / 接口入口）

表现层负责接收外部请求（HTTP / RPC / 消息等）、进行基础解析与校验，然后将请求转交给应用层处理，并将结果映射为协议级响应。

---

## 一、职责与边界

- **协议适配，不做业务决策**
  - 仅负责：解析请求、基础参数校验、鉴权结果接收、调用应用服务、组装响应；
  - 不直接实现业务规则、不直接访问数据库。

---

## 二、请求与响应处理

- **请求解析与校验**
  - 明确区分：必填参数、可选参数与默认值；
  - 可将复杂校验提取为独立的验证模块或使用统一验证框架。

- **响应结构统一**
  - 响应体结构应遵循项目约定（如 `code`、`message`、`data`）；
  - 错误码与错误消息需与项目统一错误码规范对齐。

---

## 三、安全与鉴权

- **认证与鉴权**
  - 使用统一的认证中间件/机制，不在控制器中手工编写重复逻辑；
  - 控制器只依赖「权限检查结果」，不实现权限判断细节。

- **防护措施**
  - 对重要接口考虑限流、防重放与基础输入过滤；
  - 对上传内容进行大小与类型限制。

---

## 四、日志与审计

- **请求级日志**
  - 在入口层记录必要的请求信息（如 TraceId、用户ID、关键参数摘要、耗时等）；
  - 避免记录敏感信息或完整大体积 payload。

- **审计日志**
  - 对关键操作（如资金变动、权限变更）调用专门的审计记录逻辑。
