# 后端适配层编码规范（Adapter / Gateway）

适配层负责与外部世界交互，将内部领域模型与外部协议/数据模型进行转换，并隔离外部系统的不稳定性。

---

## 一、职责与边界

- **对接外部系统**
  - 与第三方 API、支付网关、消息中间件、平台 SDK 等交互；
  - 不直接承载业务决策，将外部响应转译为内部可理解的结果。

- **适配与防腐**
  - 将外部模型映射为内部 DTO 或领域对象，避免外部类型在内部广泛传播；
  - 为易变的外部依赖提供防腐层（Anti-Corruption Layer）。

---

## 二、接口与错误处理

- **封装外部调用细节**
  - 对上层暴露稳定的接口（如 `chargePayment`、`sendNotification`），内部可更换具体实现或供应商；
  - 对外部错误进行分类并转换为内部可识别的错误类型或错误码。

- **重试与降级**
  - 明确重试策略、超时设置与降级策略；
  - 避免在调用方自行编写重复的重试/降级逻辑。

---

## 三、监控与审计

- **调用链监控**
  - 对关键外部调用记录耗时、成功率、错误类型等指标；
  - 集成到统一监控体系与告警规则中。

- **审计要求**
  - 对涉及安全、资金或敏感操作的外部调用保留必要的审计信息。
